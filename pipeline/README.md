# situator — конвейер объединения сработок в инциденты

Реализация конвейера Отчёта v3 §8. Всё в Docker; на хосте остаются только `runs/` (результаты)
и кэш извлечённых кадров `runs/cache/`. Дизайн: `docs/plans/pipeline-environment.md`.

## Быстрый старт

```bash
make build     # один раз: образ + веса YOLO11 (после этого офлайн)
make ingest    # индексация inputs/ -> runs/index.db
make info      # сводка индекса
make run       # полный прогон: карточки + метрики в runs/run_<время>/
```

Точечные прогоны (фильтры):

```bash
docker compose run --rm pipeline run --window "2026-06-03 19:00..2026-06-03 20:00" --object site-a
docker compose run --rm pipeline run --object dg --no-detector   # смоук без YOLO
docker compose run --rm pipeline bench --limit 300               # латентность стадий
```

## Результат прогона

`runs/run_*/cards.jsonl|csv|xlsx` — карточки: `incident_id`, `type`
(`реал | среда | деградация | несхлопнуто`), период, камеры, описание, `trigger_ids`.
`runs/run_*/metrics.json` — сжатие, доля авто-закрытого, аудит REAL (потери в авто-закрытых = 0).

## Конфиги (bind-mount, без пересборки)

- `config/pipeline.yaml` — пороги: детектор (модель/пер-классовые conf/imgsz), деградация,
  статик-подавление, τ_gap, τ_handoff.
- `config/cameras.yaml` — карта камер и граф соседства (пары `confirmed: false` подтвердить!).
- `config/lexicon.yaml` — словарь описаний карточек.

## Словарь описаний (`config/lexicon.yaml`)

Описание карточки (<=160 символов) — **черновик для оператора**, собирается без LLM по шаблону:

```
<Тип>: <кто, по словарю классов> — <камера(ы)>; <время суток> <период> UTC; <N сработок>[; пометки]
Движение: человек, легковая машина — Площадка A, камера 1 (двор); утром 04:01–04:48 UTC; 267 сработок; переход между камерами
```

**Порядок пополнения словаря:**
1. **Новый класс детектора** (добавили в `pipeline.yaml: detector.classes`) → добавить русское
   имя в `classes:` и позицию в `class_order:` (порядок вывода: люди → животные → техника).
   Если класс — суррогат (лось детектится как `horse`), в имени ставим знак вопроса:
   `крупное животное (лошадь?)` — оператор понимает, что это догадка.
2. **Новая пометка** (причина «несхлопнуто», тип шторма и т.п.) → ключ в `extras:`;
   код передаёт ключ, словарь решает формулировку.
3. **Тон/терминология заказчика** (например «ТС» вместо «машина») → правится только yaml,
   код и пересборка не нужны: конфиг монтируется bind'ом.
4. Принцип: словарь говорит **только то, что детектор реально видел** (классы, камеры, время,
   счёт). Интерпретации («разгрузка», «обход территории») в словарь не добавляем — это уровень
   VLM-описаний (отложенный шаг), их место в поле `description` то же.

## Что сознательно отложено (v0)

- Пер-камерный классификатор «реал/шум» — ждёт ответа о происхождении меток FALSE (Отчёт v3 §7.3).
- Видео-гейтинг ночных кандидатов (несколько кадров клипа + motion) — после калибровки v0.
- Re-ID в межкамерной склейке — пока только время+граф соседства.

## Замечание про GPU

В Docker на macOS GPU Apple недоступен — инференс CPU. Тот же пакет нативно (`device=mps`)
ускоряется автоматически; авторитетный перф-замер — этот же образ на A10.
