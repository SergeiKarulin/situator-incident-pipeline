# CLAUDE.md — руководство для LLM-сессии, продолжающей эту работу

Ты подхватываешь **проверенный рабочий прототип**. Здесь всё, что нужно знать, чтобы
продолжить без доступа к исходной сессии. Прочитай целиком, прежде чем менять код.

## Контекст задачи

Система видеонаблюдения генерирует тысячи сработок детектора движения в сутки; ~98% — средовой
шум (туман, блики, помехи тракта), реальных событий ~2%. Конвейер объединяет сработки в
карточки-инциденты: оператор видит ~10 карточек/час вместо ~39, ничего реального не теряется.
Метрики последнего полного прогона — в README и `docs/design.md`.

## Данные: где и в каком формате (подробно — docs/DATA_FORMAT.md)

- Выгрузка кладётся в `./inputs` хоста (маппится в `/data/inputs` контейнера, read-only).
- Структура: папки-страницы с подпапками сработок `<trigger_id>/<YYYYMMDD>/`, внутри —
  `*_clip.mp4` (главный вход), `*_list.txt` (из него regex `/recordings/(cam\d+)/` даёт
  камеру), опционально jpg.
- Рядом — xlsx-отчёт со статусами (лист «Список сработок»).
- **Вход конвейера — только видео.** jpg в выгрузках встречаются, но игнорируются
  (`use_jpgs: false`): в проде их нет, и они смещены (приложены только к обработанным
  сработкам — для обучения непригодны).

## Запуск

```bash
cp pipeline/config/cameras.example.yaml pipeline/config/cameras.yaml  # заполнить под контур!
make build && make ingest && make run
```

Результаты в `runs/`. Конфиги bind-mount — менять пороги можно без пересборки; пересборка
нужна только при изменении кода пакета.

## Инварианты — не нарушай без перепроверки

1. **Аудит REAL — блокирующий критерий.** После каждого изменения порогов/логики гони полный
   прогон и проверяй `metrics.json: real_lost_in_autoclosed == []`. Этот аудит уже ловил
   регресс: «улучшение» порога boat с 0.45 до 0.65 потеряло 2 реальных события (техника под
   чехлом детектится YOLO как boat). Не поднимай `conf_per_class.boat`.
2. **4 позиции кадров (0.5/4/9/12 с) — все обязательны.** Кадр в конце клипа ловит объекты,
   входящие в кадр к концу записи; без него теряется REAL (проверено на реальном кейсе).
3. **ffmpeg может молча не отдать кадр** (exit 0, файла нет) при seek за конец видеопотока —
   поэтому в `frames.py` каскад позиций с проверкой существования файла. Не упрощай.
4. **xlsx выгрузки — inlineStr без sharedStrings**; парсер `situator/xlsx.py` (zipfile+ElementTree)
   проверен на этих файлах. Не заменяй на openpyxl не перепроверив.
5. **`static_suppression.min_span_hours: 4` не снижать**: машина, приехавшая на 46 минут, —
   событие, а не мебель сцены. `never_suppress` (люди, животные) не сокращать.
6. **Правило подтверждённого ядра** в `sessions.py`: «реал» требует ≥2 активных сработок рядом
   во времени ЛИБО объект в ≥2 кадрах одной сработки. Без него один ложный блоб (person 0.36
   на мокром асфальте) склеивает 2.5 часа дождливой ночи в гигантский ложный «реал».
7. Сработки **без кадров** идут оператору («несхлопнуто»), а не в «среду»: закрывать молча то,
   что никто не видел, нельзя.
8. **Пороги в `pipeline.yaml` валидны только для контура, на котором калибровались.**
   Новый контур/новые камеры → обязательная перекалибровка по процедуре из `docs/design.md`
   («Перекалибровка на новом контуре»); приёмка — пустой `real_lost_in_autoclosed` на
   размеченной выборке нового контура. Не переноси числа слепо.
9. Docker на macOS = CPU-only (Apple GPU в Linux-VM не пробрасывается). Тот же пакет нативно
   сам выберет `mps`. Часовые метрики снимай с **холодным** кэшем кадров (`runs/cache/frames`) —
   иначе займёшь извлечение из замера и получишь приукрашенные цифры.

## Карта кода (pipeline/situator/)

| Модуль | Ответственность |
|---|---|
| `ingest.py` + `xlsx.py` | выгрузка → SQLite-индекс (`runs/index.db`) |
| `frames.py` | кадры из клипа (ffmpeg, кэш) |
| `degradation.py` | эвристика помех тракта (доля насыщенных строк + мерцание тона) |
| `detector.py` | YOLO11, пер-классовые пороги, счёт кадров с объектом |
| `staticmap.py` | подавление статичных объектов (IoU-кластеры, класс-группы матчинга) |
| `sessions.py` | темпоральные сессии, правило ядра, шторма деградации |
| `crosscam.py` | межкамерная склейка по графу соседства (union-find) |
| `cards.py` | карточки + описания по словарю (`config/lexicon.yaml`, ≤160 симв.) |
| `metrics.py` | сжатие, доли, аудит REAL |
| `cli.py` | оркестрация: `situator ingest|run|bench|info` |

Пополнение словаря описаний — `pipeline/README.md`, раздел «Словарь описаний».

## Дальнейшие работы (приоритезирует команда; обоснования — в документе-предложении)

1. **API/стриминговый режим** — необходимое условие боевой работы. Батч уже эквивалентен
   реплею потока: держи открытые сессии в памяти, закрытие по таймауту τ_gap → публикация
   карточки. Пик потока 0.75 сработки/с — один узел.
2. Пер-камерный классификатор «реал/шум» — добивает остаточный шум. Прежде чем учить на
   статусах из xlsx, выясни их происхождение (могли быть проставлены авто-фильтром, а не
   человеком — тогда это дистилляция чужих ошибок).
3. Видео-гейтинг сомнительных кандидатов (motion-анализ всего клипа).
4. Зоны интереса / события-состояния (открывание ворот сейчас уходит в «среду»).
5. Re-ID в межкамерной склейке; VLM-описания (поле `description` совместимо).

## Стиль работы, который здесь принят

- Любое изменение порогов — с числовым обоснованием (замер на реальных кадрах, см. комментарии
  в `pipeline.yaml` — там зафиксированы калибровки).
- Сначала смоук на коротком окне (`--window`), потом полный прогон.
- Не заявлять «работает» без прогона и метрик в том же сообщении.
- Данные заказчика (видео, кадры, имена объектов/камер) не публиковать: боевой `cameras.yaml`
  в git не попадает (см. `.gitignore`).
