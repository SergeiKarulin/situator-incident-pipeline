# Локальное окружение конвейера объединения сработок (M5, Docker)

Дата: 2026-06-10. Статус: каркас v0, готов к приёму полной суточной выгрузки.

## Цель

Когда придёт полная выгрузка сработок за 1–2 суток — прогнать её через конвейер из Отчёта v3 §8
локально на Apple M5 48GB и получить **упрощённые карточки инцидентов** (номер, тип, краткое
описание, id всех объединённых сработок) + метрики качества и скорости. Всё в Docker, на хосте
не остаётся ничего, кроме файлов репозитория и папки `runs/`.

## Ключевые решения

1. **Всё в Docker** (требование). Следствие, о котором надо помнить: Docker на macOS работает в
   Linux-VM и **не видит GPU Apple (MPS)** — инференс в контейнере только CPU. Для разработки и
   качества это ок (объёмы — тысячи кадров, не real-time). Когда понадобится скорость:
   (а) перф-замер на арендованной A10 тем же образом, (б) опциональный нативный запуск пакета
   на хосте через venv в папке проекта (тот же код, device=mps) — venv одной папкой, удаляется
   `rm -rf`, мусора не оставляет. По умолчанию — только Docker.
2. **Один образ, один CLI** (`situator`). Никаких микросервисов: конвейер — пакетная обработка,
   оркестрация стадий внутри CLI. Compose нужен только для монтирования томов и удобных команд.
3. **Детектор — YOLO11 (ultralytics), веса зашиты в образ** на этапе сборки (`yolo11s.pt` для CPU-
   разработки; модель меняется в конфиге, для A10/MPS — `yolo11x.pt`). После сборки образ офлайн.
4. **Хранилище индекса — SQLite** в `runs/index.db`: переживает перезапуски контейнера, нулевая
   инфраструктура, удобно дебажить с хоста.
5. **xlsx читаем/пишем штатно** (pandas+openpyxl): ограничение «нет openpyxl» относилось к хосту,
   в контейнере ставим что хотим.
6. **Классификатор «реал/шум» — код есть, обучение заблокировано** до выяснения происхождения
   меток FALSE (Отчёт v3 §7.3). Конвейер v0 работает без него: детектор + деградация + сессии.
7. **Граф соседства камер — конфиг** `pipeline/config/cameras.yaml`, заполнен по текущим знаниям
   (пары соседних камер заполняются по контуру заказчика).

## Конвейер (стадии = Отчёт v3 §8)

```
ingest:   выгрузка (папки сработок + xlsx) ──► SQLite-индекс (сработка: id, utc, объект,
          камера из list.txt, статус, кадры, клип)
run:
  [0.5]   деградация сигнала (OpenCV-эвристика по кадру: насыщенные горизонтальные полосы +
          вертикальная частота тона) ──► шторм деградации схлопывается в 1 карточку/час/камеру
  [1]     детектор человек/ТС (YOLO11, до 3 кадров на сработку; кадра нет — извлечь из клипа
          ffmpeg-каскадом -ss 2/0.5/0) ──► activity | environment
  [2]     темпоральные сессии внутри (объект, камера): пауза > τ_gap (150 с деф.) = новая сессия
  [3]     межкамерная склейка: сессии соседних камер (по графу), пересечение/handoff ≤ τ_handoff
          (120 с деф.), обе activity ──► один инцидент
cards:    карточки JSONL + CSV + XLSX: id, type (реал|среда|деградация), период, камеры,
          описание (шаблон), trigger_ids; html-обзор с миниатюрами
metrics:  сжатие (сработок→карточек, видимых→видимых), потерянные REAL (цель 0 на эталоне),
          доля авто-закрытого, сравнение с реестром групп v2 (если xlsx дан)
bench:    реплей потока по таймстампам с замером лат./пропускной (p50/p95, сработок/с)
```

## Форматы

Карточка (`runs/<run>/cards.jsonl`):

```json
{"incident_id": 17, "type": "реал", "period_utc": ["2026-06-04T01:28:41", "2026-06-04T01:33:08"],
 "object": "site-b", "cameras": ["cam3"], "n_triggers": 9,
 "description": "Активность (person): cam3, 9 сработок, 01:28–01:33 UTC",
 "trigger_ids": ["5980027", "5980029", "..."]}
```

`type`: `реал` (есть объект интереса) · `среда` (авто-закрыто, аудируемая корзина) ·
`деградация` (шторм помех камеры) · `несхлопнуто` (низкая уверенность — на ручной разбор).

## Структура в репо

```
docker-compose.yml, Makefile
pipeline/
  Dockerfile  pyproject.toml  README.md
  config/cameras.yaml  config/pipeline.yaml
  situator/  (xlsx.py ingest.py degradation.py detector.py frames.py
              sessions.py crosscam.py cards.py metrics.py replay.py cli.py)
runs/        (выходы прогонов; в контейнере /runs)
```

Тома: `./inputs → /data/inputs:ro`, `./runs → /runs`, конфиги bind'ом (правки без пересборки).

## Что считаем успехом v0 (на текущих данных, до полной выгрузки)

- `ingest` собирает индекс по всему `inputs/` (≈7.8k папок + 8 692 строки xlsx);
- окно E (туман) → 1–2 карточки «среда», 0 «реал»;
- окно D (помехи) → карточка «деградация» (шторм) + НЕ потерять 2 REAL полностью молча
  (ожидаемо: v0 без видео-гейтинга может их не поднять — это зафиксированный в Отчёте v3 §6.2
  предел стоп-кадров; в карточках они обязаны остаться хотя бы внутри шторма, не «в никуда»);
- окно B (гаражи) → межкамерный «реал»-инцидент двух соседних камер (склейка по графу);
- окно A (минивэн) → 1–2 «реал»-сессии вместо 58 групп;
- метрики печатаются, REAL-аудит работает.

## Когда придёт полная выгрузка

1. Положить её рядом (любая папка), `make ingest DATA=<путь>`.
2. `make run` → карточки + метрики. Калибровка порогов по `config/pipeline.yaml`.
3. Учить пер-камерный классификатор — только после ответа о происхождении меток (§7.3).
4. Перф-замер: `make bench` (CPU-числа — ориентир); авторитетные цифры — тот же образ на A10.

## Перекалибровка на новом контуре (обязательна при переносе)

Все пороги `pipeline.yaml` откалиброваны на исходном контуре (9 камер). Перенос на новый
контур без перекалибровки даст непредсказуемые метрики. Процедура:

1. **Собрать выгрузку нового контура** за >=1 сутки (захватить день и ночь) + xlsx со
   статусами, если есть. `make ingest`.
2. **Деградация тракта**: выбрать по 30-40 сработок с «помеховых», «туманных» и «чистых»
   часов; снять распределения `score_image` (sat_rows_frac, hue_flicker) и поставить пороги
   между распределениями с запасом (как это делалось — комментарий в `pipeline.yaml`,
   блок degradation).
3. **Пер-классовые conf**: смоук-прогоны часовых окон (`run --window`), разобрать ложные
   «реал» и потерянные REAL по `metrics.json` и карточкам; двигать пороги только с числовым
   обоснованием. Помнить кейс boat: техника под чехлом детектится «не своим» классом —
   прежде чем поднимать порог класса, проверить, чьи REAL на нём держатся.
4. **Статик-карта**: параметры обычно переносимы; проверить, что «мебель» (припаркованные
   машины и т.п.) попала в static_clusters лога, а событийные объекты — нет.
5. **Граф соседства и timezones** — заполнить cameras.yaml под контур.
6. **Приёмка**: полный прогон, `real_lost_in_autoclosed == []` на размеченной выборке нового
   контура + выборочный визуальный аудит карточек двух интервалов (день + ночь).
