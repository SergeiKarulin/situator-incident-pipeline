# Формат входных данных

Конвейер ест выгрузку сработок из системы видеонаблюдения (Интеллект/Ситуатор).
Данные кладутся в **`./inputs`** в корне репозитория (в контейнер маппится как
`/data/inputs`, read-only). Путь можно поменять в `docker-compose.yml`.

## Структура каталогов

```
inputs/
├── <страница выгрузки>/            # любое имя; либо сразу папки сработок (плоско)
│   ├── <trigger_id>/               # имя = числовой id сработки (incident_id)
│   │   └── <YYYYMMDD>/
│   │       ├── <id>_<YYYYMMDDTHHMMSS>Z_clip.mp4   # ГЛАВНЫЙ ВХОД: клип сработки (~10–15 c)
│   │       ├── <id>_<YYYYMMDDTHHMMSS>Z_list.txt   # пути записей: /recordings/camNN/...
│   │       └── *.jpg                              # опционально; конвейером ИГНОРИРУЮТСЯ
│   └── ...
├── report_<даты>.xlsx              # отчёт со статусами (см. ниже)
└── ...
```

Принимается и плоская структура: `inputs/<trigger_id>/<YYYYMMDD>/...` без страниц.

Что откуда берётся:

| Поле | Источник | Примечание |
|---|---|---|
| `trigger_id` | имя папки | = `incident_id` в xlsx |
| Время (UTC) | xlsx `incident_date`; фолбэк — таймстамп в имени файла клипа | |
| Камера | `*_list.txt`, regex `/recordings/(cam\d+)/` | надёжнее, чем `sensor_name` в xlsx (заполняется частично) |
| Объект | xlsx `object_name` | |
| Статус | xlsx `status` | используется только для аудита качества |

## xlsx-отчёт

Лист **«Список сработок»**, по строке на сработку. Колонки: `incident_id`, `incident_date`
(UTC, `YYYY-MM-DD HH:MM:SS.ffffff`), `object_name`, `sensor_name`, `category`, `status`
(`REAL` / `FALSE` / `NOT_PROCESSED`), `action` (`MD_START`), `employee`, `note`,
`confirm_date`, `process_date`.

Технически файл пишет строки как **inlineStr** (sharedStrings отсутствует) — штатный парсер
`situator/xlsx.py` это учитывает. Если отчёта нет, конвейер работает только по папкам
(время из имён файлов), но аудит REAL будет недоступен.

## Требования к клипам

- Контейнер mp4 (H.264/H.265), любое разрешение (проверено на 1280×720 и 1920×1080).
- Длительность ~10–15 с. Кадры извлекаются на позициях 0.5/4/9/12 с; для коротких клипов
  работает каскад-фолбэк. Известная особенность: видеопоток бывает короче длительности
  контейнера — извлечение это учитывает.
- На кадрах допустим текстовый оверлей (дата/время/имя точки) — он не мешает.

## Конфиг камер (обязателен)

`pipeline/config/cameras.yaml` (создать из `cameras.example.yaml`): соответствие
`camNN → объект + имя`, **граф соседних камер** (без него межкамерная склейка не работает)
и часовые пояса объектов (для «утром/ночью» в описаниях карточек).
