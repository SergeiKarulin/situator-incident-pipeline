# situator-incident-pipeline

Конвейер объединения сработок видеонаблюдения в инциденты. Превращает сырой поток сработок
детектора движения (id + видеоклип) в короткий список **карточек-инцидентов** с типом,
русскоязычным описанием и списком объединённых сработок.

Проверен на полной реальной выгрузке (4 суток, 8 692 сработки, 9 камер, 6 объектов):

| Метрика | Результат |
|---|---|
| Сжатие потока | 8 692 сработки → 1 058 карточек (**8.2×**) |
| Видимая нагрузка оператора | **~10 карточек/час** против ~39/час у штатной группировки |
| Авто-закрыто как шум/помехи | 44.4% сработок — в аудируемых корзинах, не удаляется |
| Межкамерные инциденты | 61 (штатная группировка не умеет вовсе) |
| Потеряно размеченных реальных событий | **0 из 178** |
| Скорость (CPU, без GPU) | 1.21 с/сработку, ×30 реального времени |

**Работаете через Claude Code / LLM-агента? Начните с [CLAUDE.md](CLAUDE.md)** — там контекст,
инварианты и грабли. Формат входных данных — [docs/DATA_FORMAT.md](docs/DATA_FORMAT.md),
архитектура и решения — [docs/design.md](docs/design.md).

## Быстрый старт

```bash
# 1. Данные: положите выгрузку в ./inputs (структура — docs/DATA_FORMAT.md)
# 2. Конфиг камер: скопируйте пример и заполните под свой контур
cp pipeline/config/cameras.example.yaml pipeline/config/cameras.yaml

make build     # docker-образ; веса YOLO11 зашиваются при сборке, дальше офлайн
make ingest    # индексация выгрузки -> runs/index.db
make info      # сводка индекса
make run       # полный прогон: карточки + метрики в runs/run_<время>/
```

Точечные прогоны и бенчмарк:

```bash
docker compose run --rm pipeline run --object site-a --window "2026-06-03 19:00..2026-06-03 20:00"
docker compose run --rm pipeline run --no-detector --limit 500   # смоук без YOLO
docker compose run --rm pipeline bench --limit 300               # латентность стадий
```

## Что на выходе

`runs/run_*/cards.jsonl|csv|xlsx` — карточки:

```json
{"incident_id": 17, "type": "реал", "object": "site-a", "cameras": ["cam1", "cam2"],
 "period_utc": ["2026-06-02 10:45:03", "2026-06-02 10:48:25"], "n_triggers": 5,
 "description": "Движение: легковая машина — Площадка A, камера 1 (двор) + Площадка A, камера 2 (ворота); днём 10:45–10:48 UTC; 5 сработок; переход между камерами",
 "trigger_ids": ["5935176", "..."]}
```

Типы: `реал` (подтверждённое движение: человек/техника/животное) и `несхлопнуто` (нужен взгляд
человека) — оператору; `среда` (средовой шум) и `деградация` (шторм помех видеотракта) —
авто-закрытые аудируемые корзины. `runs/run_*/metrics.json` — сжатие, доля авто-закрытого и
**аудит REAL** (ни одна размеченная реальная сработка не должна попасть в авто-закрытые —
блокирующий критерий качества).

## Архитектура (кратко)

```
сработка (id + clip.mp4) → кадры (ffmpeg, 4 позиции) → детектор деградации сигнала
  → YOLO11 (14 классов, пер-классовые пороги) → подавление статичных объектов сцены
  → темпоральные сессии (правило подтверждённого ядра) → межкамерная склейка по графу соседства
  → карточки + описания по словарю (config/lexicon.yaml, ≤160 символов, без LLM)
```

Все пороги — `pipeline/config/pipeline.yaml` (bind-mount: правка без пересборки образа).
Обоснование каждого решения и калибровок — `docs/design.md` и комментарии в конфиге.

## Ограничения текущей версии

- Батч-режим (обработка выгрузки); стриминговый приём — первоочередная доработка
  (батч-логика эквивалентна реплею потока, см. CLAUDE.md §«Дальнейшие работы»).
- Остаточные одиночные ложные «реал» в сложных ночных условиях (туман в подсветке).
- События-состояния (открывание ворот) уходят в «среду» — объектный детектор их не видит.
- В Docker на macOS инференс CPU-only (GPU Apple в контейнеры не пробрасывается).
- **Пороги откалиброваны на конкретном контуре (9 камер).** На другом наборе камер метрики
  поплывут — перед боевым применением пройдите перекалибровку (процедура: `docs/design.md`,
  раздел «Перекалибровка на новом контуре»; контроль — аудит REAL).

Лицензионное замечание: используется Ultralytics YOLO11 (AGPL-3.0) — учитывайте при выборе
модели распространения.
